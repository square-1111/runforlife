# Privacy Policy — RunForLife

**Last updated:** March 25, 2026

## Overview

RunForLife is a personal fitness tracking application built for private use by its developers. This policy describes how we handle data collected through the Garmin Connect API.

## Data Collection

RunForLife accesses the following data from Garmin Connect for authorized users only:

- **Activity data** — running, walking, and workout activities (distance, duration, pace, heart rate)
- **Health metrics** — VO2 max, HRV, training status, training load
- **Sleep data** — sleep duration, stages, and sleep score
- **Daily statistics** — steps, resting heart rate, stress, body battery
- **Body composition** — weight (if available via Garmin scale)

## Data Usage

All data is used **exclusively** for:

- Personal training analysis and coaching
- Generating AI-powered training recommendations
- Tracking fitness goals (e.g., running frequency, VO2 max improvement)

## Data Storage

- Data is stored **locally** on the users' own devices
- No data is stored on external servers or cloud infrastructure
- Authentication tokens are cached locally for API access and are not shared

## Data Sharing

- **We do not sell, share, or distribute any user data to third parties**
- **We do not use data for advertising or marketing purposes**
- Data is accessible only to the explicitly authorized users of the application

## Users

This application serves **2 users** (the developers themselves). There is no public-facing service or user registration.

## Data Retention

Users can delete all locally stored data at any time by removing the application's data directory. No data persists beyond the user's local storage.

## Security

- Garmin credentials are stored in environment variables, not in source code
- Authentication tokens are cached locally with restricted file permissions
- The application source code is open source for transparency
